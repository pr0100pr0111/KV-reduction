# Voice Data Redaction Service - Backend

Это бэкенд-сервис для нашего хакатон-проекта по анонимизации голосовых данных. Написан на Go.

## Основные функции

*   REST API для управления задачами обработки.
*   Прием аудиофайлов (WAV, MP3, OGG, FLAC).
*   Оркестрация: вызывает Python-сервис для тяжелой работы (STT, NER).
*   Хранение информации о задачах (пока что в памяти).

## Запуск и Установка

### Требования

*   Go (версия 1.22 или выше)

### Шаги для запуска

1.  **Настройка окружения:**

    Для локального запуска нужно создать файл с переменными окружения. Просто скопируй пример:
    ```bash
    cp .env.example .env
    ```
    Внутри `.env` можно поменять порт (`PORT`) или адрес AI-сервиса (`AI_SERVICE_URL`), если нужно. По умолчанию сервер стартует на порту `8080`.

2.  **Установка зависимостей:**

    Перейди в папку `backend/` и выполни команду:
    ```bash
    go mod tidy
    ```

3.  **Запуск сервера:**

    Из той же папки `backend/` выполни:
    ```bash
    go run ./cmd/server
    ```
    Если все ок, в консоли появится сообщение `🚀 Backend starting on port 8080`.

## Как тестировать (API)

Для тестов можно использовать `curl` или любой другой HTTP-клиент (Postman, Insomnia).

### 1. Проверка доступности (Health Check)

```bash
curl http://localhost:8080/health
```

### 2. Загрузка аудиофайла

Создай любой тестовый файл (например, `touch test.mp3`) и выполни команду, подставив свой путь к файлу:

```bash
curl -X POST -F "audio=@test.mp3" http://localhost:8080/api/v1/upload
```

В ответе получишь `job_id`. **Сохрани его**, он понадобится для следующих запросов.

### 3. Получение статуса задачи

Замени `YOUR_JOB_ID` на ID из предыдущего шага.

```bash
curl http://localhost:8080/api/v1/job/YOUR_JOB_ID
```

### 4. Получение списка всех задач

```bash
curl http://localhost:8080/api/v1/jobs
```

### 5. Удаление задачи

```bash
curl -X DELETE http://localhost:8080/api/v1/job/YOUR_JOB_ID
```
